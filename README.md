# MUD-WoW

一个以 Vanilla 1.12 为资料基线、从普通人类冒险者视角重走暴风王国早期危机的文字 RPG。

项目当前处于设计与技术验证阶段。首个目标不是复刻完整 World of Warcraft，而是完成北郡 1–5 级垂直切片，验证纯文字探索、任务、连续自动战斗与低频决策、成长和多人共享状态是否成立。

## 当前决策

- 使用 TypeScript 实现模块化单体权威服务端。
- 多个客户端通过双向连接进入同一个服务端世界。
- 服务端负责裁决任务、战斗、掉落、交易和持久状态。
- 首发以 Web 客户端为主，关系型数据库首选 PostgreSQL。
- 首版不引入 Rust、Tauri、微服务、Redis 或分布式事件总线。
- AI 只用于文字表现，不决定任务完成、伤害、掉落或剧情事实。
- 规则型队友控制器与暂离接管使用确定性战斗规则，不由生成式 AI 操作。

## 下一里程碑

完成北郡垂直切片：

1. 至少两个客户端可以同时连接并看见共享房间状态；
2. 玩家可以移动、查看环境并完成一条有持久状态的任务；
3. 战士和法师拥有可区分的基础战斗节奏；
4. 两名玩家可以参与同一场战斗并获得一致的事件结果；
5. 退出并重新连接后恢复服务端当前权威状态；战斗中断线不会回滚到断线瞬间。

实现范围和验收口径以 [产品范围与 MVP](docs/product-scope.md) 为准。
