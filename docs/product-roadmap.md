# 产品路线图：从北郡切片到暴风王国首章

> 状态：门槛驱动的产品路线图  
> 更新日期：2026-07-17  
> 上游范围：[产品范围](./product-scope.md)  
> 设计依据：[调研与 MVP 蓝图](./research/vanilla-wow-text-mud-research.md)、[人类路线参考](./research/vanilla-wow-human-route-reference.md)、[战斗系统设计](./combat-system.md)、[系统架构](./architecture.md)与[多人一致性](./multiplayer-consistency.md)

本文从现有 A–E 阶段中整理出当前四版本候选窗口，覆盖 A–D；阶段 E 的第二阵营仍保留在本窗口之后。`V1`–`V4` 只是路线图顺序，不是 SemVer、数据库迁移号或 `content_version`；实际构建和内容包仍使用各自独立的版本标识。

当前只有 V1 的玩法与验收已经冻结。V2–V4 是候选方向，只有在前一版本通过、对应版本契约完成并冻结后，才转为实现承诺。

## 1. 路线总览

```text
V1 北郡垂直切片
  → V2 迪菲亚闭环
      → V3 暴风城政治续篇
          → V4 横向世界扩展（赤脊山 / 暮色森林二选一）
```

| 版本 | 当前状态 | 等级与地区 | 对玩家的核心承诺 | 退出门槛 |
|---|---|---|---|---|
| V1：北郡垂直切片 | **当前执行；设计已冻结** | 1–5；北郡 | 证明文字探索、低注意力战斗、成长、双人共享状态和可靠恢复能够形成完整体验 | 北郡 `00`–`06` 全部实现，`07` 的 P0、体验与真人试玩门槛通过 |
| V2：迪菲亚闭环 | **候选；待 V1 后冻结** | 1–22；北郡、艾尔文森林、精简暴风城、西部荒野、死亡矿井 | 从普通冒险者成长为能够完成五人副本的队伍，击败范克里夫并完成第一条有起因、调查和副本高潮的完整故事 | 四职业、跨区域成长、五人队和死亡矿井形成可重复完成的闭环，并留下 `The Unsent Letter` 钩子 |
| V3：暴风城政治续篇 | **候选；待 V2 后冻结** | 22–31；暴风城监狱、军情七处、宫廷相关区域 | 兑现未寄出的信，把迪菲亚事件从地方危机推进到暴风城政治后篇 | 续篇长链无软锁、无提前剧透，监狱与既有组队系统稳定复用，政治篇获得明确收束 |
| V4：横向世界扩展 | **条件式候选** | 赤脊山 15–25，或暮色森林 18–30 | 根据真实玩家偏好增加一条独立于迪菲亚的可选区域篇章 | 只选择一个区域，并证明纳入该版本的故事弧独立、可追溯且可以完整体验 |

本路线图不承诺日历日期。仓库目前只有设计基线，没有可用于估算的实现吞吐、内容生产速度和缺陷数据；V1 完成后再用真实数据为 V2 排期。

## 2. 版本晋级规则

每个版本依次经过：

```text
候选 → 范围冻结 → 实现中 → 候选构建 → 验收通过
```

晋级必须遵守以下规则：

1. **上一版本先通过。** 部分模块可运行、内容已录入或单条路径可玩，都不能替代完整版本验收。
2. **先冻结规则，再实现。** V2–V4 启动前都要建立自己的切片契约、玩家流程、内容、战斗／系统变化、多人恢复和验收基线；未写入规则所有者的能力不进入版本。
3. **估算不等于承诺。** 研究稿中的房间、任务、敌人、装备、专业和 Boss 数量只用于早期容量判断，必须在对应版本设计包中重新核对。
4. **旧版本必须持续回归。** 冻结 V1 `content_version` fixture 上的 V1 全套验收必须继续通过；V2 当前内容包另做 V1→V2 迁移和兼容回归，不能把 V1 的两人上限、南路边界或简化 `recover` 错套到 V2 新语义上。
5. **公开发行另设门槛。** “首个可玩发行版”表示产品闭环完成，不表示已经获准公开分发；品牌、素材、许可证和知识产权审查仍由[产品范围的发布边界](./product-scope.md#9-版权与发布边界)单独决定。

## 3. V1：北郡垂直切片

### 3.1 版本目标

用一次约 60–90 分钟的真实流程，证明这款游戏最基础的假设成立；假设本身与验收口径见[北郡 v1 设计包](./northshire-v1/README.md)。

### 3.2 冻结范围摘要

以下只用于说明路线图体量，不拥有精确内容清单或玩法语义；实现与验收始终以[北郡 v1 设计包](./northshire-v1/README.md)为唯一权威。

- 1–5 级北郡，从修道院到南路边界，不进入闪金镇；
- 11 个房间、10 条任务、6 类敌人模板、20 个固定敌人出生点；
- 8 个个人采集节点、7 名北郡功能 NPC 和 1 个不可生成的外部 NPC 引用；
- 12 个 ability、27 个 item；
- 战士、法师两个职业，完成训练后各有 5 个当前可用动作；
- 最小 Web 文字终端、任务／知识状态、战斗、个人掉落、成长、装备、训练和商店；
- 两名玩家进入同一个 `CombatSession`，至少 5 个客户端可用于多人验收组合；
- 死亡 `recover`、断线 `afk_profile` 接管、重连快照、进程崩溃恢复和幂等奖励；
- 在南路首次幂等写入 `sliceCompleteAt`，`Report to Goldshire` 保持 `active`。

### 3.3 通过条件

- [北郡 v1 设计包](./northshire-v1/README.md)的 `00`–`06` 全部实现；
- [验收与试玩](./northshire-v1/07-acceptance-and-playtest.md)要求的自动化、集成、真实客户端 E2E 和真人试玩四层证据全部为 `PASS`；`BLOCKED` 与 `NOT_RUN` 均不计为通过；
- P0 包括但不限于主任务软锁、角色串档、重复经验／任务进度／物品／金币、旧连接越权和永久状态丢失均为 0；
- 不能把双人战斗、断线接管、重连或崩溃恢复拆到 V2 后再补。

### 3.4 明确不做

闪金镇内部与后续地区、盗贼／牧师、正式队伍、五人副本、规则型队友、竞争拾取、交易、完整跑尸、生成式 AI 表现和公开运营均不进入 V1。

## 4. V2：迪菲亚闭环——首个真正可玩的发行版

### 4.1 版本目标

让新角色能够完成 `北郡 → 艾尔文森林 → 西部荒野 → 死亡矿井` 的 1–22 级旅程。故事从北郡地方治安逐步过渡到迪菲亚调查，在范克里夫战后获得明确阶段结局，并以 `The Unsent Letter` 留下下一版本入口。

V2 可以有内部里程碑，但只有整个闭环通过时才能对外称为 V2：

1. **内容管线里程碑：** 沿用稳定内容 ID；如新增顶层 zone Schema，在 V2 契约中冻结并迁移，同时完成跨区域移动、艾尔文、必要的暴风城服务节点及西部荒野内容。V2 契约必须定义 `Report to Goldshire` 的真实交付路径，以及既有 `sliceCompleteAt` 的兼容和迁移语义；路线图不预选最终状态方案，但迁移不得重复奖励或破坏 V1 fixture 回归。
2. **四职业与小队里程碑：** 在战士、法师之外加入盗贼、牧师；完成正式队伍、治疗仇恨、控制、驱散、打断、标记和集火。盗贼先验证能量、连击点和潜行，牧师与正式队伍一起验证跨玩家治疗和法力规划。
3. **副本里程碑：** 完成死亡矿井实例、五人上限、手工邀请、亲自到达入口、需求／贪婪、固定 Boss 蓝装、团灭／重置和规则型 `CompanionController`。

内部里程碑不是删减版发行版本：艾尔文或西部荒野内容可供开发测试，但没有死亡矿井结局时，V2 仍未完成。

### 4.2 候选纳入项与启动时必须冻结的议题

- 艾尔文森林 5–10/11 的矿洞、鱼人／失踪卫兵、农场、迪菲亚和豺狼人任务簇；
- 与主线、训练和服务直接相关的精简暴风城节点，不一次制作完整主城；
- 西部荒野 10–18/20 的地方生活、人民军和迪菲亚调查；赤脊山只允许保留调查链必需的联络节点，不把区域主体并入 V2；
- 死亡矿井入口、实例与范克里夫结局；
- 战士、法师、盗贼、牧师四职业，每职业的准确动作、训练和成长在 V2 战斗包中冻结；
- 正式队伍、五人副本、队长粗粒度战术、真实玩家暂离接管和确定性规则型队友；
- Boss 公开意图、阶段和资源压力；规格定义了响应窗口的关键机制从公开到结算不少于 5 秒，超时按已配置的默认反应继续，不使用地形、视线或精确距离；
- V2 契约必须冻结跨区域所需的最小旅行与死亡循环；旅店绑定／炉石、必要的区域交通、墓地与尸体恢复、耐久和修理是研究候选，未冻结前不自动成为退出门槛；
- 多 WorldInstance／副本实例仍由同一个权威应用管理，不因加入副本而提前拆成微服务。

### 4.3 通过条件

- 四职业都能独立完成野外内容，进入队伍后职责和资源节奏可辨识；
- 玩家仅凭文字地标即可完成至少 90% 的纳入任务，不依赖外部地图；
- 迪菲亚真相按任务与知识状态逐步揭示，不在北郡或艾尔文开场提前解释范克里夫；
- 每个死亡矿井 Boss 至少提供两个可读、可应对的机制；规格定义了响应窗口的关键机制从公开到结算不少于 5 秒，超时按默认反应继续；
- 单人规则型队友队伍与真人五人队遵守相同技能、资源、接战、仇恨和 Boss 规则；规则型队友不能读取隐藏阶段或零延迟反应，1–3 名真人采用混编队友还是动态缩放由 V2 契约冻结；
- 队伍、实例和竞争拾取分别复用既有序列化键、事务、唯一约束、幂等结果和逐 scope 快照协议；共享槽位最终至多生成一个物品实例，一个角色最终至多属于一个普通队伍，提交后响应丢失或重启均不重复结果；
- 主动暂离的 `manual` 与意外断线的 `disconnect` 保持不同来源：重连只自动清除 `disconnect`，`manual` 必须显式解除；`controlEpoch` 不得被误当作 profile 切换原因，queued／started 动作语义继续成立；
- 冻结 V1 `content_version` fixture 上的 V1 全套验收继续为 `PASS`；代表性 V1 存档升级到 V2 后，`sliceCompleteAt`、#54、旧幂等记录和首次真实交付符合 V2 契约，重复资产为 0；
- 从空库安装和从代表性 V1 数据库升级均成功，备份恢复、健康检查、优雅停机、结构化日志及关键队列／事务／连接指标可用；延迟和容量 SLO 等 V1 实测后再冻结；
- 范克里夫战后主篇明确结束，`The Unsent Letter` 可被可靠获取。

### 4.4 明确不做

- 不制作完整暴风城、赤脊山主体、暮色森林、圣骑士、术士或第二阵营；
- 不做自动匹配、自动传送、地形／视线战斗、PvP、40 人团队和 1–60 世界；
- 研究稿中的精简专业、完整交易、邮件、公会和拍卖生态不作为 V2 退出门槛；若以后要做，必须单独进入某一版本契约，不能作为“首发常识”被静默加入。

## 5. V3：暴风城政治续篇

### 5.1 版本目标

兑现范克里夫掉落的未寄出信，把地方叛乱推进为暴风城监狱、军情七处和贵族政治的同一矛盾后篇。V3 应以复用 V2 能力和增加高密度叙事内容为主，而不是再建设一套核心玩法平台。

### 5.2 范围

- 22–31 级；
- `The Unsent Letter` → `Bazil Thredd` → `The Stockade Riots` → 调查／潜入 → `The Attack!` → `An Audience with the King` 的完整续篇；
- 暴风城监狱、军情七处、宫廷及任务实际需要的城区；
- 经任务链核对后确实需要的调查、潜入和袭击事件，继续由任务图、知识状态和结构化 encounter 表达；
- 复用 V2 的队伍、副本、仇恨、拾取和故障恢复，不为监狱另造第二套规则。

### 5.3 进入与通过条件

进入 V3 前必须同时满足：

- V2 的迪菲亚闭环和未寄出信掉落已经通过验收；
- 玩家反馈表明需要继续追踪同一矛盾，而不是优先要求新的横向区域；
- V2 的任务知识、五人副本和跨区域内容管线已经稳定。

V3 通过时必须证明：

- 续篇前置、知识揭示、任务物品和奖励全程无软锁、无串写、无越级剧透；
- 暴风城监狱能够复用既有队伍和实例语义；
- Vanilla 事实保持正确，包括由卡特拉娜·普瑞斯托接待最终觐见，不提前改写成后期时间线；
- 政治篇获得明确结局，且不依赖赤脊山或暮色森林主体才能完成。

### 5.4 明确不做

不同时制作赤脊山和暮色森林，不填满暴风城所有城区与 NPC，不提前展开高等级奥妮克希亚内容，也不把各地危机强行解释为同一幕后黑手。

## 6. V4：横向世界扩展槽

V4 是发布顺序上的第四个版本，不代表角色必须在 31 级后进入。它增加的是一条与现有等级带交叠的替代区域，立项时只选择一个方向：

| 候选 | 适合的玩家需求 | 内容边界 | 主要复用／新增重点 |
|---|---|---|---|
| 赤脊山，15–25 | 更需要战争前线、城镇防务和连续野外作战 | 黑石兽人、豺狼人、鱼人、物资危机和湖畔镇求援；不是迪菲亚支线，也不提前改写为黑龙阴谋 | 复用队伍、护送、防守和跨区通信；重点验证多敌群与地方防务节奏 |
| 暮色森林，18–30 | 更需要文字调查、恐怖气氛和彼此独立的短篇 | 守夜人、斯塔文、狼人、亚伯克隆比、摩拉迪姆等故事群；不制造统一总反派。31–35 级终章留待后续，或在 V4 契约中明确标记为等级压缩改编 | 复用任务知识层，按需要增加局部持久世界事件；重点验证线索、误导和环境叙事 |

选择依据应来自 V2/V3 的真人试玩、任务完成／放弃数据、导航求助、玩家偏好和实际内容生产速度。没有证据时不并行制作两区。

V4 通过时，所选区域必须：

- 拥有自己的进入、推进、高潮和离开路径；
- 仅凭文字地标可理解，任务事实和改编来源可追溯；
- 不把跨区通信误写为共同幕后黑手；
- 复用既有权威状态、战斗和恢复机制，新增规则只服务于该区域真实需要。

另一候选区域、杜隆塔尔第二阵营、圣骑士／术士和 1–60 世界继续留在 V4 之后，不自动进入范围。

## 7. 跨版本不变量

### 7.1 产品与内容

- 以 Vanilla 1.12／Classic 旧世界资料为主要事实冻结点；`original`、`adapted` 和 `new` 必须可区分、可追溯。
- 战斗保持连续自动执行、约 5 秒摘要、持久战术和一次性覆盖；普通战允许短暂不输入，当前遭遇结束后停止。
- 不引入地形、视线、精确码数、朝向或现代地图箭头来代替文字地标设计。
- 生成式 AI 只能作为可关闭的表现层，不能决定剧情事实、任务完成、技能选择、伤害、掉落或永久状态。

### 7.2 架构与一致性

- 在没有 [ADR-0001 重新评估触发条件](./adr/0001-typescript-modular-monolith.md#重新评估触发条件)所列的证据前，保持 Node.js + TypeScript 模块化单体、单一权威应用、PostgreSQL 和 HTTP + WebSocket；出现触发条件后仍须先证明单体内优化不足，并完成跨边界失败、重试和幂等设计。
- 内容使用稳定 ID；同一构建使用唯一、统一包级且不可变的 `content_version`，内容变更不得复用旧版本，破坏性 ID 变更必须有显式迁移。新区域以新增数据为主，不复制一套任务或战斗业务代码。
- 同一作用域保持单写者和确定性排序；客户端只提交意图，不能裁决移动、伤害、奖励或任务状态。
- 命令幂等身份保持 `(actorId, commandId)`；重试复用原 envelope，`controlEpoch` 阻止旧连接，事件使用 `(scopeEpoch, scope, serverSeq)`，完整快照携带逐 scope `scopeCursors[]`。新增队伍、区域和副本 scope 复用同一协议，不另造单数全局游标。
- 每个角色只有一个逻辑权威 `CharacterState`；战斗投影不得拥有或整体反写经验、任务、背包、装备和金币，奖励账本与队伍聚合也不是第二份角色余额。永久结果先事务提交再广播，命令重试、断线、重连和崩溃不得制造重复或回滚。
- 规则型 `CompanionController` 与真人使用相同战斗规则，且不属于生成式 AI。

### 7.3 质量门槛

- 每个版本都必须有规则所有者、自动化、集成、真实客户端 E2E 和真人试玩证据；验收文档不能反向发明玩法。
- 任何新版本的 P0 都至少包括：主线无软锁、角色无串档、永久奖励零重复、旧连接零越权、故障恢复结果唯一。
- 延迟、并发人数和在线规模目标应由 V1 实测数据决定；在没有数据前不虚构 SLO，也不以“未来扩展”作为提前拆服务的理由。

## 8. 当前执行顺序

1. 只实现并验收 V1，不为 V2 预建正式组队、规则型队友、交易或副本平台。
2. V1 全部通过后，先复盘代码变更量、内容录入速度、测试缺陷和真人试玩数据，再冻结 V2 的精确数量与规则。
3. V2 完成后，以未寄出信的续篇需求决定是否启动 V3。若 V3 进入条件不成立而偏好明确指向横向区域，应先更新本路线图和版本编号，不能静默跳过 V3 或自动启动 V4。
4. V4 只在真实偏好数据足以支持“赤脊山或暮色森林”二选一时立项。
