## 2. 依赖与阅读顺序

按编号阅读和冻结：

```text
00 切片契约
  -> 01 玩家流程
      -> 02 世界与内容
          -> 03 任务状态包
              -> 04 战斗与成长
                  -> 05 战术构筑与关键决策
                      -> 06 多人与恢复
                          -> 07 验收与试玩
```

依赖规则：

1. 上游文档决定“要做什么”和“规则是什么”；下游文档只能细化接口、测试和证据。
2. `07` 不创造玩法规则。若测试无法写出唯一预期结果，应回到拥有该决定的上游文档补齐。
3. 修改上游规则时，必须检查所有下游文档和对应验收用例；不能只改实现。
4. 同一决定只在一个文档中拥有正文，其他文档以链接或验收 ID 引用，避免多份相互漂移的定义。
5. 某个依赖文档尚未存在、未冻结或相互冲突时，相关实现与验收结论不得标记为完成。

## 4. 本包完成条件

北郡 V1 设计包只有同时满足以下条件才算完整：

- 00–07 全部存在并标明状态；
- 每项首切片规则有且只有一个所有者文档；
- 每项首切片目标至少映射到一种测试证据；
- 所有 P0 验收门槛有可重复执行的步骤与量化结果；
- 真人试玩报告包含构建版本、内容版本、测试组合、原始指标、失败原因和后续动作；
- 当前规则中不存在未回填的玩法决定或因此阻塞的用例；
- 没有以“待决定”规则为前提却被标记为通过的测试。
